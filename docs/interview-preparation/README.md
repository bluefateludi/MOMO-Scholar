# MOMO Scholar 面试导向学习记录

## 目标

这套文档以通过技术面试为最终目标，但不以背诵标准答案为主要方式。学习过程必须结合 MOMO Scholar 的真实代码，深入理解模块原理、设计取舍、边界情况和生产化方案，最后再将理解转化为清晰的面试表达。

开发规范由仓库根目录的 `AGENTS.md` 管理；这里不规定开发流程。

## 学习方法

每个核心模块按照以下三层整理：

1. **项目实践**：解决什么问题，数据如何流动，对应哪些代码和测试。
2. **深入原理**：底层机制、复杂度、边界情况、替代方案和技术取舍。
3. **面试转化**：简短项目表达、递进式问题以及生产化改进。

一个知识点需要依次达到：

- 知道：能说明它是什么。
- 理解：能解释它如何工作。
- 应用：能定位项目中的实现和测试。
- 比较：能分析替代方案及取舍。
- 表达：能结合项目承受连续追问。

## 学习节奏

- Task 完成后：记录涉及的模块、关键设计和待深入问题。
- Chunk 完成后：系统学习核心原理并完成小实验。
- 每两个 Chunk：进行一次模拟面试，记录薄弱点。
- 面试前：从模块文档提炼项目介绍和高频问题。
